FILE SYSTEM AND LINK ANALYSIS REPORT

Commands Used:
touch file1
ln file1 hardlink
ln -s file1 softlink
ls -li
stat file1
stat hardlink
stat softlink
rm file1
ls -li

Inode Information:
The original file and hard link shared the same inode number.
The symbolic link had a different inode number.

Comparison:
Hard Link:
- Shares the same inode.
- Remains accessible after deleting the original file.

Symbolic Link:
- Has a separate inode.
- Becomes broken after deleting the original file.

Metadata:
Metadata was collected using the stat command.

Conclusion:
Hard links reference the same physical data blocks, whereas symbolic links only store the pathname of the target file.

FILE ACCESS AND I/O INVESTIGATION REPORT

Commands Used:
lsof | head
ulimit -a
echo "Hello Linux" > output.txt
ls abc 2> error.txt
echo "Second Line" >> output.txt

Open Files:
The lsof command displayed currently opened files and their associated processes.

File Descriptors:
Each open file is associated with a file descriptor maintained by the Linux kernel.

Output Redirection:
The output of echo was successfully redirected to output.txt.

Error Redirection:
The error generated by "ls abc" was redirected to error.txt.

Resource Limits:
The ulimit command displayed the system resource limits such as open files, memory size, stack size, and CPU time.

Conclusion:
Linux manages file input/output using file descriptors. Standard output and standard error can be redirected independently for better process control.
